# 网络粘贴板前端开发文档（Tauri + Vue 3 + TypeScript）
本文档基于需求文档和 MySQL 服务端，提供 **Tauri 前端** 完整开发流程，包含项目初始化、核心功能实现、跨端适配、与服务端对接等，确保前端支持密码认证、文本同步、历史管理等核心功能。

## 一、前置环境准备
### 1. 核心依赖安装
#### （1）Node.js 环境（18+）
Tauri 前端基于 Node.js 构建，需先安装：
```bash
# 下载 Node.js 18+（推荐用 nvm 管理版本）
# macOS/Linux 安装 nvm：
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install 18 && nvm use 18

# 验证版本
node -v  # 需 ≥ 18.0.0
npm -v   # 需 ≥ 9.0.0
```

#### （2）Tauri 开发依赖
Tauri 需 Rust 环境（已在服务端步骤中安装），再安装 Tauri CLI：
```bash
# 全局安装 Tauri CLI（用于创建/打包项目）
npm install -g @tauri-apps/cli@2.0.0  # 适配 Tauri 2.x（稳定版）

# 验证 Tauri 环境（自动检查依赖是否齐全）
tauri dev --help  # 无报错则环境正常
```

#### （3）可选工具（提升开发效率）
- 代码编辑器：VS Code + 插件（Tauri、Vue Language Features、Tailwind CSS IntelliSense）
- 移动端调试：iOS 需 Xcode（macOS 专属），Android 需 Android Studio

### 2. 核心工具：加密工具类（composables/useCrypto.ts）
与服务端加密算法一致（bcrypt 密码哈希、AES-256-GCM 内容加密）：
```typescript
import bcrypt from 'bcryptjs';
import aesjs from 'aes-js';
import { Base64 } from 'js-base64';

// AES-256-GCM 配置（与服务端一致）
const AES_KEY_LENGTH = 32; // 256 位
const GCM_IV_LENGTH = 12;  // GCM 模式推荐 IV 长度
const GCM_TAG_LENGTH = 16; // 标签长度

/**
 * 密码哈希（bcrypt，与服务端一致）
 * @param password 明文密码
 * @returns 哈希后的密码
 */
export const hashPassword = async (password: string): Promise<string> => {
  const salt = await bcrypt.genSalt(12); // 工作因子=12（服务端一致）
  return bcrypt.hash(password, salt);
};

/**
 * 验证密码（bcrypt）
 * @param password 明文密码
 * @param hash 数据库中的密码哈希
 * @returns 是否匹配
 */
export const verifyPassword = async (password: string, hash: string): Promise<boolean> => {
  return bcrypt.compare(password, hash);
};

/**
 * 从密码派生 AES 密钥（服务端相同逻辑）
 * @param password 明文密码
 * @returns 32 位 AES 密钥
 */
const deriveAesKey = async (password: string): Promise<Uint8Array> => {
  // 简化实现：实际可使用 PBKDF2 加盐派生（更安全）
  const encoder = new TextEncoder();
  const data = encoder.encode(password);
  // 用 SHA-256 哈希密码，得到 32 位密钥
  const hashBuffer = await crypto.subtle.digest('SHA-256', data);
  return new Uint8Array(hashBuffer);
};

/**
 * AES-256-GCM 加密文本
 * @param text 明文文本
 * @param password 密码（派生密钥）
 * @returns 加密后的数据（Base64 编码：IV+密文+标签）
 */
export const encryptText = async (text: string, password: string): Promise<string> => {
  const key = await deriveAesKey(password);
  const iv = crypto.getRandomValues(new Uint8Array(GCM_IV_LENGTH)); // 随机 IV
  const encoder = new TextEncoder();
  const data = encoder.encode(text);

  // 初始化 AES-GCM 加密器
  const aesGcm = new aesjs.ModeOfOperation.gcm(key, iv);
  const encrypted = aesGcm.encrypt(data);

  // 拼接 IV + 密文 + 标签（服务端需按此解析）
  const combined = new Uint8Array([...iv, ...encrypted, ...aesGcm.getAuthTag()]);
  return Base64.encode(combined);
};

/**
 * AES-256-GCM 解密文本
 * @param encryptedBase64 加密后的 Base64 字符串
 * @param password 密码（派生密钥）
 * @returns 明文文本
 */
export const decryptText = async (encryptedBase64: string, password: string): Promise<string> => {
  const key = await deriveAesKey(password);
  const combined = Base64.decodeUint8Array(encryptedBase64);

  // 拆分 IV、密文、标签
  const iv = combined.subarray(0, GCM_IV_LENGTH);
  const encrypted = combined.subarray(GCM_IV_LENGTH, combined.length - GCM_TAG_LENGTH);
  const tag = combined.subarray(combined.length - GCM_TAG_LENGTH);

  // 初始化 AES-GCM 解密器
  const aesGcm = new aesjs.ModeOfOperation.gcm(key, iv);
  aesGcm.setAuthTag(tag);
  const decrypted = aesGcm.decrypt(encrypted);

  const decoder = new TextDecoder();
  return decoder.decode(decrypted);
};
```

### 3. 状态管理：Pinia 配置（pinia/ 目录）
#### （1）认证状态（pinia/auth.ts）
```typescript
import { defineStore } from 'pinia';
import { getToken, setToken, removeToken } from '@/utils/storage';

export const useAuthStore = defineStore('auth', {
  state: () => ({
    token: getToken() || '', // JWT Token（本地存储）
    isLogin: !!getToken(),  // 是否登录
    password: '',           // 临时存储明文密码（用于加密）
    autoLock: true,         // 是否自动锁定
    lockDuration: 5,        // 自动锁定时长（分钟）
  }),
  actions: {
    // 登录（存储 Token 和密码）
    login(token: string, password: string) {
      this.token = token;
      this.password = password;
      this.isLogin = true;
      setToken(token); // 存储到 Tauri 本地存储
    },
    // 退出登录
    logout() {
      this.token = '';
      this.password = '';
      this.isLogin = false;
      removeToken(); // 清除本地存储
    },
    // 修改密码
    updatePassword(newPassword: string) {
      this.password = newPassword;
    },
  },
});
```

#### （2）服务端配置（pinia/config.ts）
```typescript
import { defineStore } from 'pinia';
import { getConfig, setConfig } from '@/utils/storage';

// 默认配置
const DEFAULT_CONFIG = {
  serverUrl: 'http://127.0.0.1:3000', // 服务端默认地址
  autoSync: true,                     // 自动同步
  darkMode: 'system',                 // 深色模式（system/light/dark）
};

export const useConfigStore = defineStore('config', {
  state: () => ({
    config: { ...DEFAULT_CONFIG, ...getConfig() }, // 合并本地存储配置
  }),
  actions: {
    // 更新配置
    updateConfig(newConfig: Partial<typeof DEFAULT_CONFIG>) {
      this.config = { ...this.config, ...newConfig };
      setConfig(this.config); // 保存到本地存储
    },
    // 重置配置
    resetConfig() {
      this.config = { ...DEFAULT_CONFIG };
      setConfig(this.config);
    },
  },
});
```

### 4. 接口封装：与服务端对接（api/ 目录）
#### （1）请求基础配置（api/request.ts）
```typescript
import axios from 'axios';
import { useAuthStore } from '@/pinia/auth';
import { useConfigStore } from '@/pinia/config';
import { notification } from '@tauri-apps/api';

// 创建 axios 实例
const service = axios.create({
  timeout: 10000, // 超时时间
});

// 请求拦截器：添加 JWT Token
service.interceptors.request.use(
  (config) => {
    const authStore = useAuthStore();
    const configStore = useConfigStore();
    // 设置服务端基础地址
    config.baseURL = configStore.config.serverUrl;
    // 添加认证 Token
    if (authStore.token) {
      config.headers['Authorization'] = `Bearer ${authStore.token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

// 响应拦截器：统一错误处理
service.interceptors.response.use(
  (response) => response.data,
  (error) => {
    // 错误提示
    notification.show({
      title: '操作失败',
      body: error.response?.data?.msg || '网络异常，请检查连接',
    });
    return Promise.reject(error);
  }
);

export default service;
```

#### （1）登录页（views/Login.vue）
支持首次设置密码、登录、配置服务端：
```vue

      </button>

      <!-- 密码输入框 -->
      <div class="mb-4">

          {{ isFirstUse ? '设置初始密码' : '输入密码' }}
        </label>
        <div class="relative">
          <input
            v-model="password"
            type="password"
            :placeholder="isFirstUse ? '设置≥6位密码' : '输入密码登录'"

            @input="checkPasswordLength"
          />
          <button @click="showPassword = !showPassword" class="absolute right-3 top-1/2 -translate-y-1/2">
            {{ showPassword ? '隐藏' : '显示' }}
          </button>
        </div>
        <p v-if="password.length > 0 && password.length < 6" class="text-red-500 text-sm mt-1">
          密码长度需≥6位
        </p>
      </div>

      <!-- 二次确认密码（首次设置） -->

        <input
          v-model="confirmPassword"
          type="password"
          placeholder="再次输入密码"
          class="w-full"
        />
        <p v-if="confirmPassword !== password" class="text-red-500 text-sm mt-1">
          两次密码不一致
        </p>
      </div>

      <!-- 操作按钮 -->
      <button
        @click="handleSubmit"
        class="w-full py-2 mt-4"
        :disabled="!isSubmitEnabled"
      >
        {{ isFirstUse ? '完成设置' : '登录' }}
      </button>

      <!-- 忘记密码（重置） -->
      <button @click="showReset = true" class="w-full mt-2 text-sm text-gray-500 hover:text-blue-600">
        忘记密码？
      </button>
    </div>

    <!-- 服务端配置弹窗 -->
    <teleport to="body">
      <div v-if="showConfig" class="fixed inset-0 bg-black/50 flex items-center justify-center p-4">
        <div class="bg-white dark:bg-gray-800 rounded-xl p-6 w-full max-w-md">
          <h3 class="text-lg font-bold mb-4">服务端配置</h3>
          <input
            v-model="serverUrl"
            type="text"
            placeholder="输入服务端地址（如 http://192.168.1.100:3000）"
            class="w-full mb-4"
          />
          <div class="flex gap-2">
            <button @click="testServerConnection" class="flex-1">测试连接</button>
            <button @click="saveServerConfig" class="flex-1">保存</button>
            <button @click="showConfig = false" class="bg-gray-300 text-gray-900 hover:bg-gray-400">取消</button>
          </div>
        </div>
      </div>
    </teleport>
  </template>

<script setup lang="ts">
import { ref, onMounted } from 'vue';
import { useAuthStore } from '@/pinia/auth';
import { useConfigStore } from '@/pinia/config';
import { setPassword, login } from '@/api/auth';
import { notification } from '@tauri-apps/api';
import axios from 'axios';

const authStore = useAuthStore();
const configStore = useConfigStore();

// 状态
const password = ref('');
const confirmPassword = ref('');
const showPassword = ref(false);
const showConfig = ref(false);
const showReset = ref(false);
const isFirstUse = ref(false); // 是否首次使用（无密码）
const serverUrl = ref(configStore.config.serverUrl);
const isSubmitEnabled = ref(false);

// 初始化：检查是否首次使用（服务端是否有密码）
onMounted(async () => {
  try {
    // 尝试访问服务端，判断是否已有密码
    await axios.get(`${configStore.config.serverUrl}/api/auth/check`);
    isFirstUse.value = false;
  } catch (error) {
    isFirstUse.value = true;
  }
});

// 检查密码长度，启用提交按钮
const checkPasswordLength = () => {
  if (isFirstUse.value) {
    isSubmitEnabled.value = password.value.length >= 6 && password.value === confirmPassword.value;
  } else {
    isSubmitEnabled.value = password.value.length >= 6;
  }
};

// 提交（设置密码/登录）
const handleSubmit = async () => {
  try {
    if (isFirstUse.value) {
      // 首次设置密码
      await setPassword(password.value);
      notification.show({ title: '成功', body: '密码设置成功，即将登录' });
    }
    // 登录（获取 Token）
    const res = await login(password.value);
    authStore.login(res.token, password.value);
    notification.show({ title: '成功', body: '登录成功' });
  } catch (error) {
    console.error('提交失败:', error);
  }
};

// 测试服务端连接
const testServerConnection = async () => {
  try {
    await axios.get(`${serverUrl.value}/health`);
    notification.show({ title: '成功', body: '服务端连接正常' });
  } catch (error) {
    notification.show({ title: '失败', body: '服务端不可用，请检查地址' });
  }
};

// 保存服务端配置
const saveServerConfig = () => {
  configStore.updateConfig({ serverUrl: serverUrl.value });
  showConfig.value = false;
  notification.show({ title: '成功', body: '服务端配置已保存' });
};
</script>

<template>
  <div class="flex flex-col h-screen p-4">
    <!-- 顶部导航栏 -->
    <header class="flex justify-between items-center mb-4">
      <h1 class="text-xl font-bold">网络粘贴板</h1>
      <div class="flex items-center gap-2">
        <span class="text-sm" :class="syncStatus === 'success' ? 'text-green-500' : 'text-orange-500'">
          {{ syncStatusText }}
        </span>
        <button @click="goToSettings" class="p-2">设置</button>
      </div>
    </header>

    <!-- 输入区域 -->
    <div class="mb-6 flex-1 flex flex-col">
      <label class="text-sm text-gray-500 dark:text-gray-400 mb-1">
        输入或粘贴内容（{{ textLength }}/100000）
      </label>
      <textarea
        v-model="content"
        class="flex-1 resize-none p-4"
        placeholder="在此输入或粘贴文本..."
        @input="checkTextLength"
      ></textarea>
      <div class="flex gap-2 mt-2">
        <button @click="copyToLocalClipboard" class="flex-1">复制到本地粘贴板</button>
        <button @click="clearContent" class="bg-gray-300 text-gray-900 hover:bg-gray-400">清空</button>
        <button @click="syncContent" class="flex-1" :disabled="isSyncing">
          {{ isSyncing ? '同步中...' : '同步到服务端' }}
        </button>
      </div>
    </div>

    <!-- 历史记录区域 -->
    <div class="h-1/3 border-t pt-4 overflow-y-auto">
      <h2 class="text-lg font-semibold mb-2">历史记录（{{ historyList.length }}条）</h2>
      <div v-if="historyList.length === 0" class="text-center text-gray-500 py-4">
        暂无历史记录
      </div>
      <div class="space-y-2">
        <div 
          v-for="item in historyList" 
          :key="item.id"
          class="p-3 bg-gray-100 dark:bg-gray-800 rounded-lg flex flex-col"
        >
          <div class="flex justify-between items-start">
            <span class="text-sm text-gray-500">
              {{ formatTime(item.timestamp) }}
            </span>
            <div class="flex gap-1">
              <button @click="copyHistoryToLocal(item.content)" class="text-sm p-1">复制</button>
              <button @click="deleteHistory(item.id)" class="text-sm p-1 text-red-500">删除</button>
            </div>
          </div>
          <p class="mt-1 text-sm line-clamp-2">
            {{ item.decryptedContent }}
          </p>
        </div>
      </div>
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted, computed } from 'vue';
import { useAuthStore } from '@/pinia/auth';
import { useConfigStore } from '@/pinia/clipboard';
import { syncClipboard, getClipboardHistory, deleteClipboard } from '@/api/clipboard';
import { encryptText, decryptText } from '@/composables/useCrypto';
import { clipboard } from '@tauri-apps/api';
import { format } from 'date-fns';

const authStore = useAuthStore();
const clipboardStore = useClipboardStore();

// 状态
const content = ref('');
const isSyncing = ref(false);
const syncStatus = ref<'idle' | 'success' | 'error'>('idle');
const historyList = ref<Array<{
  id: number;
  content: string; // 加密后的内容
  timestamp: number;
  decryptedContent: string; // 解密后的内容
}>>([]);

// 文本长度计算
const textLength = computed(() => content.value.length);

// 同步状态文本
const syncStatusText = computed(() => {
  switch (syncStatus.value) {
    case 'success': return '同步成功';
    case 'error': return '同步失败';
    default: return '未同步';
  }
});

// 初始化：加载历史记录
onMounted(async () => {
  await fetchHistoryList();
  // 自动同步（若开启）
  if (clipboardStore.config.autoSync) {
    await syncContent();
  }
});

// 检查文本长度（≤10万字符）
const checkTextLength = () => {
  if (content.value.length > 100000) {
    content.value = content.value.slice(0, 100000);
  }
};

// 同步内容到服务端
const syncContent = async () => {
  if (!content.value.trim()) {
    notification.show({ title: '提示', body: '内容不能为空' });
    return;
  }

  isSyncing.value = true;
  syncStatus.value = 'idle';
  try {
    // 加密内容
    const encryptedContent = await encryptText(content.value, authStore.password);
    // 上传到服务端（时间戳用当前毫秒数）
    await syncClipboard(encryptedContent, Date.now());
    syncStatus.value = 'success';
    notification.show({ title: '成功', body: '内容同步成功' });
    // 重新加载历史记录
    await fetchHistoryList();
  } catch (error) {
    syncStatus.value = 'error';
    console.error('同步失败:', error);
  } finally {
    isSyncing.value = false;
  }
};

// 获取历史记录（并解密）
const fetchHistoryList = async () => {
  try {
    const res = await getClipboardHistory();
    if (res.success && res.data) {
      // 解密每条记录
      const decryptedList = await Promise.all(
        res.data.map(async (item: any) => ({
          id: item.id,
          content: item.content,
          timestamp: item.timestamp,
          decryptedContent: await decryptText(item.content, authStore.password),
        }))
      );
      // 按时间戳倒序排序
      historyList.value = decryptedList.sort((a, b) => b.timestamp - a.timestamp);
    }
  } catch (error) {
    console.error('获取历史记录失败:', error);
  }
};

// 复制到本地粘贴板
const copyToLocalClipboard = async () => {
  await clipboard.writeText(content.value);
  notification.show({ title: '成功', body: '已复制到本地粘贴板' });
};

// 复制历史记录到本地
const copyHistoryToLocal = async (encryptedContent: string) => {
  const decrypted = await decryptText(encryptedContent, authStore.password);
  await clipboard.writeText(decrypted);
  notification.show({ title: '成功', body: '已复制到本地粘贴板' });
};

// 删除历史记录
const deleteHistory = async (id: number) => {
  try {
    await deleteClipboard(id);
    notification.show({ title: '成功', body: '记录已删除' });
    // 重新加载历史
    await fetchHistoryList();
  } catch (error) {
    console.error('删除失败:', error);
  }
};

// 格式化时间戳
const formatTime = (timestamp: number) => {
  return format(new Date(timestamp), 'yyyy-MM-dd HH:mm:ss');
};

// 跳转到设置页
const goToSettings = () => {
  router.push('/settings');
};
</script>
```

### 6. 路由配置（router/index.ts）
```typescript
import { createRouter, createWebHistory } from 'vue-router';
import { useAuthStore } from '@/pinia/auth';
import Login from '@/views/Login.vue';
import Home from '@/views/Home.vue';
import Settings from '@/views/Settings.vue';

const router = createRouter({
  history: createWebHistory(),
  routes: [
    { path: '/', redirect: '/home' },
    { path: '/login', component: Login, meta: { requiresAuth: false } },
    { path: '/home', component: Home, meta: { requiresAuth: true } },
    { path: '/settings', component: Settings, meta: { requiresAuth: true } },
    { path: '/:pathMatch(.*)*', redirect: '/home' }, // 404 跳转
  ],
});

// 路由守卫：未登录跳转到登录页
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore();
  if (to.meta.requiresAuth && !authStore.isLogin) {
    next('/login');
  } else {
    next();
  }
});

export default router;
```


## 五、跨端适配与打包
### 1. 本地运行与调试
```bash
# 进入前端项目根目录（clipboard_frontend）
cd clipboard_frontend

# 启动开发模式（桌面端，实时热更新）
tauri dev
```
- 开发模式下，前端代码修改后会自动刷新，可直接调试与服务端的对接逻辑。
- 移动端调试需额外配置（见下文）。

### 2. 桌面端打包（Windows/macOS/Linux）
```bash
# 1. 先构建 Vue 前端静态资源
cd www
npm run build
cd ..

# 2. 打包桌面端安装包
tauri build
```
- 打包产物默认在 `src-tauri/target/release/bundle/` 目录：
    - Windows：`.exe` 安装包
    - macOS：`.dmg` 镜像文件
    - Linux：`.deb`（Debian/Ubuntu）或 `.rpm`（Fedora）

### 3. 移动端打包（iOS/Android）
#### （1）iOS 打包（需 macOS + Xcode）
```bash
# 1. 初始化 iOS 项目
tauri ios init

# 2. 构建 iOS 应用（生成 .ipa 文件）
tauri ios build
```
- 需在 Xcode 中配置签名证书，才能安装到真机或上传 App Store。

#### （2）Android 打包（需 Android Studio）
```bash
# 1. 初始化 Android 项目
tauri android init

# 2. 构建 Android 应用（生成 .apk/.aab 文件）
tauri android build
```
- 需在 Android Studio 中配置签名密钥，适配不同 Android 机型。


## 六、常见问题排查
### 1. 跨域问题
服务端已配置 `CorsLayer::permissive()`（开发环境），若仍报错：
- 检查前端 `tauri.conf.json` 中 `http.scope` 是否包含服务端地址；
- 生产环境建议服务端配置严格的 CORS 规则（仅允许前端 App 域名）。

### 2. 加密不一致
前后端加密/解密失败，需确保：
- AES 算法参数一致（IV 长度 12 位、标签长度 16 位、密钥 256 位）；
- bcrypt 工作因子一致（均为 12）；
- 密码派生密钥逻辑一致（前端用 SHA-256，服务端需相同）。

### 3. 移动端权限问题
- iOS：需在 `src-tauri/ios/Info.plist` 中添加网络权限（`NSAppTransportSecurity`）；
- Android：需在 `src-tauri/android/app/src/main/AndroidManifest.xml` 中添加网络权限（`android.permission.INTERNET`）。

### 4. 本地存储异常
Tauri `plugin-store` 存储失败，检查：
- `tauri.conf.json` 中 `allowlist.store` 已启用；
- 存储路径是否有读写权限（桌面端默认在系统用户目录，移动端在应用沙盒）。
